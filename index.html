Visitas:

El objetivo principal del OVI (Objeto Virtual de Información) es fomentar la educación virtual y el aprendizaje autónomo por parte del estudiante, permitiendo tener una plataforma de fácil manejo e intuitiva que permita finalizar positivamente el curso de Diseño de Sitios Web.

Objetivos:

Entre sus objetivos específicos, aprenderemos cómo gestionar un proyecto Web a través de los repositorios GIT y GITHUB, aplicando correctamente el uso de control de cambios y publicaciones. También produciremos contenidos web bajo la estandarización HTML5 y CSS3, como lo pueden ser Páginas Web o Sitios Web. Y por último crearemos mockups o maquetaciones necesarias para el desarrollo web, como también, el mejoraremos las técnicas de redacción y presentación de contenido digital.

Introducción:

El diseño Web debe ser considerado como un arte, ya que es una forma de expresión única, artística y llena de profundidad, que ha cambiado la forma en que las personas adquieren conocimiento y se interrelacionan, por lo tanto, un diseñador web deberá ser un artista, que deberá reflejar a través del sitio web su pensamiento, su esencia y el propósito de su creación, sin encasillarse mucho en etiquetas o formalismos al momento de desarrollar, ya que en la actualidad es posible hacer cosas inimaginables y formas de expresión que no hubiéramos podido pensar antes del inicio de este siglo. La invitación a leer el contenido de este OVI es llenar a ese artista de algunas metodologías que le harán reflejar su obra en ese lienzo llamado navegador web, iniciaremos desde el manejo de repositorios web como lo son GIT y GITHUB que nos permiten publicar contenido Online y también llevar control detallado de los cambios que sufren nuestros ficheros durante el proceso de creación. Se descubrirá el mundo del HTML (HyperText Markup Language) en su versión 5 que define los nuevos estándares de desarrollo web, viendo desde etiquetas, atributos y valores; hasta llegar visualización de los contenidos a través de los Display, generando la estructura básica de una página HTML (Head, Body). CSS3 es la última evolución del lenguaje de las Hojas de Estilo en Cascada y es la que nos permite dar una visualización agradable del contenido, más fluida y una mejora en la estética del contenido, modificando colores, tamaños de fuentes, fuentes de texto, tamaño de objetos, tratamiento a imágenes.

GIT y GIT HUB

GIT es una herramienta que nos permite controlar versiones de nuestros proyectos y trabajar de forma colaborativa y descentralizada. Es particularmente relevante para diseñadores web, ya que es en esta área donde los diseñadores trabajamos con código, con contenido de lenguajes formales como HTML, CSS y JAVASCRIPT. GitHub es una forja para alojar proyectos utilizando el sistema de control de versiones Git. Se utiliza principalmente para la creación de código fuente de programas de computadora



LINEAS DE COMANDO GIT:

- NOMBRE DE USUARIO:git config --global user.name "Tu Nombre Aquí"

- EMAIL:git config --global user.email your_email@example.com

- AGREGAR NUEVOS ARCHIVOS O CARPETAS: git add <archivo o carpeta>

- MODIFICACIONES DEL PROYECTO: git commit -m "Mensaje descriptivo de la modificación que has hecho."

- CAMBIOS EN EL REPOSITORIO: git push origin master

Es una herramienta en la cual es un sistema de control de versiones y también podemos hacer uso para trabajar remotamente con diferentes usuarios, al igual poderlo descargar en cualquier momento.

Estructuras básicas de HTML5


Empecemos con el DOCTYPE, es muy sencillo de escribir y ha sido recortado, después la etiqueta HTML la cual tiene el atributo LANG para el idioma del sitio. 

<!DOCTYPE html>
< html lang="es">
< head>

Muy sencillo para empezar ¿no?, ahora toca el turno del título, este sigue siendo el mismo, con diferencia de la etiqueta META, donde le decimos al navegador que tipo de codificado es el documento HTML, de preferencia usemos el UTF-8 que acepta nuestros acentos y ñ con mas compatibilidad. En realidad es algo demasiado corto y que no quita mucho tiempo, todo esto sigue estando en la etiqueta HEAD que no ha cambiado para nada, es algo muy parecido a XHTML, la cuestión es resumir el código.

< title> Titulo de la web</ title>
< meta charset="utf-8" />

Pasemos a las etiquetas link, son las que enlazan nuestros estilos CSS, los Favicones y los Feeds. Estas son muy cortas con excepción de el feed que sigue casi igual, solo que ahora usaremos algo que ya existia, el atributo REL que usamos en los links para cuestiones de SEO. REL es para decir, que estamos enlazando, ya que el atributo TYPE no se utiliza a menos que usemos para enlazar el FEED. 

<link rel="stylesheet" href="estilos.css" />
<link rel="shortcut icon" href="/favicon.ico" />
<link rel="alternate" title="Pozolería RSS" type="application/rss+xml" href="/feed.rss" />

Ya terminamos con la parte no visible de la página web, es el turno del cuerpo o BODY, esta etiqueta sigue siendo útil para lo mismo, pero a continuación empieza algo interesante que son las etiquetas de la estructura del sitio visual. La etiqueta HEADER es la cabecera donde va el nombre del sitio, el logotipo y descripción de la página web, ejemplo. Recomiendo que solo se use un H1 en una cabecera por cuestiones de SEO, pero es posible poner más de un H1 y HEADER. 

<body>
<header>
<h1>Mi sitio web</h1>
<p>Mi sitio web creado en html5</p>
</header>

Como pueden ver, la etiqueta HEADER es muy equivalente a crear un DIV con ID=”header”, pero ahora tenemos una etiqueta especial para ello. Pasamos con el contenido o SECTION usando H2 de titulo y el contenido en etiquetas P dentro de una etiqueta llamada ARTICLE para artículos o post, esto por cuestiones de SEO que es muy recomendable también, ejemplo. 

<section>
<article>
<h2> Titulo de contenido< h2>
<p> Contenido (además de imágenes, citas, videos etc.) < /p>
</ article>
</ section>

SECTION es el equivalente a un DIV con ID=”contenido” y ARTICLE a un DIV con ID=”post” o “articulo” Otra etiqueta interesante es la etiqueta ASIDE, lo que viene siendo nuestro SIDEBAR o barra lateral y es muy fácil de implementar con H3 de titulo y P de contenido dentro de ella. 

< aside>
< h3>Titulo de contenido< /h3>
< p>contenido< /p>
< /aside>

ASIDE es nuestro equivalente a un DIV con ID=”sidebar” o “barra” Y finalizando con el pie de la página con la etiqueta muy obvia FOOTER, un ejemplo. 

< footer> < /span< p style="padding-left: 30px;">< span style="color: #333399;"> Creado por mi el 2011< /span>< /p>
< span style="color: #333399;">
< /footer>

es un lenguaje de programación al lado del cliente su principal función es hacer visual el contenido en la página web, para se debe estudiar sus princupales caracteristicas que es la de mostrar el contenido y hacer vvusialmente.
